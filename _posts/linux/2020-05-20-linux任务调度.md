# 概述

`任务调度`是指系统在某个时间执行的特定的命令或程序。任务调度又分为系统工作，比如病毒扫描、个别用户操作，比如对mysql数据库的备份。

# 基本语法

```shell
# crontab [选项]
# 常用选项
# -e 编辑定时任务
# -l 查询crontab 任务
# -r 删除当前用户所有的crontab 任务
# service crond restart 重启调度任务
```

# 特殊符号说明

```shell
# 示例
*/1 * * * * ls -l /etc/ > /tmp/to.txt

# 参数说明
# 第一个*  一小时当中的第几分钟  0-59
# 第二个*  一天当中的第几小时    0-23
# 第三个*  一个月当中的第几天    1-31
# 第四个*  一年当中的第几天      1-12
# 第五个*  一周当中的星期几      0-7（0和7都代表星期日）
# ------------------------------------------------------
# ，代表不连续的时间
# 0 8，12，16 * * * 代表每天的8点0分，12点0分，16点0分都执行一次
# ------------------------------------------------------
# - 代表连续的时间范围
# 0 5 * * 1-6 代表周一到周六的凌晨五点执行命令
# ------------------------------------------------------
# */n  代表每隔多久执行一次
# */10 * * * * 代表每隔10分钟就执行一次命令
```

# exp

```shell
45 22 * * * # 在22点45分执行命令
0 17 * * 1 # 每周1的17点0分执行命令
0 5 1，15 * * # 每月 1号和15好凌晨5点执行命令
40 4 * * 1-5 # 每周1到周五凌晨4点40执行命令
*/10 4 * * * # 每天凌晨4点每隔十分钟执行一次
0 0 1，15 * 1 # 每月1号和15号，每周1的0点0分都会执行命令。
*/1 * * * * date >> /tmp/mydate # 每隔一分钟将当前的日期信息追加到/tmp/mydate中
```
